# csharp-style-guide

A very opinionated C# style guide that provides ways of working and style
recommendations through an **MCP server**. The guidance — Architectural Decision
Records (ADRs) and style guidelines, with a focus on C# Web APIs — lives as
Markdown in the [`designs/`](designs/) folder and is served to your AI tools over the
[Model Context Protocol](https://modelcontextprotocol.io).

The server fetches the documents from GitHub at runtime, so the guidance can be
updated in this repository without releasing a new version of the tool.

## Tools exposed

| Tool | Description |
| --- | --- |
| `list_documents` | List every ADR and guideline with its metadata. |
| `search_documents` | Search documents by keyword (id, title, summary, type, status, tags). |
| `get_document` | Return one complete document, including its full Markdown, by id. |

## Installation

The server is published to NuGet.org as a **.NET tool**. It requires the
[.NET 10 SDK](https://dotnet.microsoft.com/download) or newer.

Install it globally:

```bash
dotnet tool install --global 4dotnet-csharp-style-guide
```

Update or uninstall later with:

```bash
dotnet tool update --global 4dotnet-csharp-style-guide
dotnet tool uninstall --global 4dotnet-csharp-style-guide
```

Once installed, the command `4dotnet-csharp-style-guide` is available on your PATH.
It speaks MCP over stdio and is meant to be launched by an MCP client, not run by
hand.

> **No-install option:** with the .NET 10 SDK you can run the tool on demand with
> `dnx 4dotnet-csharp-style-guide`, which downloads and runs it without a global
> install. Use this form as the `command` in any of the configs below if you prefer
> not to install the tool.

## Adding the MCP server to your AI tool

By default the server reads this repository (`4Dotnet/csharp-style-guide`, branch
`main`). You can point it elsewhere or raise the GitHub API rate limit by setting
environment variables on the server entry:

| Variable | Default | Purpose |
| --- | --- | --- |
| `GitHub__Organization` | `4Dotnet` | Repository owner. |
| `GitHub__Repository` | `csharp-style-guide` | Repository name. |
| `GitHub__Branch` | `main` | Branch/ref to read from. |
| `GitHub__Token` | _(none)_ | GitHub token for higher rate limits / private repos. |

### Claude Code

Add the server with the CLI:

```bash
claude mcp add 4dotnet-csharp-style-guide -- 4dotnet-csharp-style-guide
```

Or add it manually to a `.mcp.json` file in your project root:

```json
{
  "mcpServers": {
    "4dotnet-csharp-style-guide": {
      "command": "4dotnet-csharp-style-guide",
      "args": []
    }
  }
}
```

Using the no-install `dnx` form instead:

```json
{
  "mcpServers": {
    "4dotnet-csharp-style-guide": {
      "command": "dnx",
      "args": ["4dotnet-csharp-style-guide"]
    }
  }
}
```

### GitHub Copilot (VS Code)

Create (or edit) `.vscode/mcp.json` in your workspace:

```json
{
  "servers": {
    "4dotnet-csharp-style-guide": {
      "type": "stdio",
      "command": "4dotnet-csharp-style-guide"
    }
  }
}
```

With optional configuration (for example, a token to raise the rate limit):

```json
{
  "servers": {
    "4dotnet-csharp-style-guide": {
      "type": "stdio",
      "command": "4dotnet-csharp-style-guide",
      "env": {
        "GitHub__Token": "${input:github_token}"
      }
    }
  }
}
```

Then open Copilot Chat in **Agent** mode and start the server from the `mcp.json`
editor; its tools become available to Copilot. The same `type: "stdio"` entry works
in a repository-level `.github/copilot/mcp.json` or Copilot's user settings.

## Repository layout

```
designs/            # the guidance content, served over MCP
  index.json        # manifest describing every document
  adr/              # Architectural Decision Records
  guidelines/       # style guidelines
src/                # the C# MCP server (.NET 10 tool)
```

## Deployment

The server is published by the [`deploy-nuget-package.yml`](.github/workflows/deploy-nuget-package.yml)
GitHub Actions workflow, which runs when changes are pushed to `main` under `src/**`
(or on manual dispatch) and pushes the package to NuGet.org. The package version is
generated by [GitVersion](https://gitversion.net/) (see [`GitVersion.yml`](GitVersion.yml)),
so every published build gets a unique version. Authentication uses
[NuGet Trusted Publishing](https://learn.microsoft.com/en-us/nuget/nuget-org/trusted-publishing)
(OIDC) from the `production` environment — no long-lived API key is stored in the
repository.
